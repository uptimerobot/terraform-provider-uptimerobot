# uptimerobot_monitor_group (Data Source)

Looks up one existing UptimeRobot monitor group by `id` or exact `name` without managing it. Monitor group names are not guaranteed unique in UptimeRobot; if more than one monitor group has the configured name, the data source returns an error and lists the matching IDs. Configure `id` to select one monitor group unambiguously.

Use this when monitor groups are managed in another Terraform state, created through the UI/API, or otherwise owned outside the current configuration. The data source exposes monitor group metadata so existing groups can be referenced from `uptimerobot_monitor.group_id` without importing them as resources.

## Example Usage

```terraform
data "uptimerobot_monitor_group" "production" {
  name = "Production"
}

resource "uptimerobot_monitor" "api" {
  name     = "Production API"
  type     = "HTTP"
  url      = "https://api.example.com/health"
  interval = 300
  group_id = tonumber(data.uptimerobot_monitor_group.production.id)
}

output "production_group_created_at" {
  value = data.uptimerobot_monitor_group.production.created_at
}
```
