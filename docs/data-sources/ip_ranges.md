# uptimerobot_ip_ranges (Data Source)

Fetches UptimeRobot monitoring IP ranges for firewall allow-lists.

The UptimeRobot Locations and IPs page documents a JSON source for monitoring IPs at `/meta/ips`. The provider derives the API host root from the configured `api_url`, so the default `https://api.uptimerobot.com/v3` reads `https://api.uptimerobot.com/meta/ips`, and an internal v3 API URL reads from the same internal host root.

## Example Usage

```terraform
data "uptimerobot_ip_ranges" "checker_europe_ipv4" {
  regions     = ["EUROPE"]
  services    = ["checker"]
  ip_versions = ["ipv4"]
}

output "uptimerobot_checker_europe_ipv4_prefixes" {
  value = data.uptimerobot_ip_ranges.checker_europe_ipv4.ipv4_prefixes
}
```

## Schema

### Optional

- `ip_versions` (Set of String) Optional IP version filter. Valid values are `ipv4` and `ipv6`. Omit to include both versions.
- `regions` (Set of String) Optional region filter. Values are matched case-insensitively against API regions such as `NORTH-AMERICA`, `EUROPE`, `ASIA`, and `OCEANIA`. Omit to include all regions.
- `services` (Set of String) Optional service filter. The current API returns `checker` entries. Omit to include all services.

### Read-Only

- `all_prefixes` (List of String) Filtered IPv4 and IPv6 CIDR prefixes.
- `create_date` (String) Creation timestamp returned by the UptimeRobot IP ranges endpoint.
- `ipv4_prefixes` (List of String) Filtered IPv4 CIDR prefixes.
- `ipv6_prefixes` (List of String) Filtered IPv6 CIDR prefixes.
- `prefixes` (Attributes List) Filtered monitoring IP prefixes with normalized CIDR and source fields. (see [below for nested schema](#nestedatt--prefixes))
- `sync_token` (String) Sync token returned by the UptimeRobot IP ranges endpoint.
