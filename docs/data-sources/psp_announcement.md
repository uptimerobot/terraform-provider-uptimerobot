# uptimerobot_psp_announcement (Data Source)

Looks up one existing UptimeRobot public status page announcement by `psp_id` plus either `id` or exact `title` without managing it. Announcement titles are not guaranteed unique within a PSP; if more than one announcement has the configured title, the data source returns an error and lists the matching IDs. Configure `id` to select one announcement unambiguously.

Use this when announcements are managed in another Terraform state, created through the UI/API, or otherwise owned outside the current configuration. The data source exposes readable announcement metadata and whether the announcement is currently pinned on its public status page.

## Example Usage

```terraform
data "uptimerobot_psp" "production" {
  name = "Example.com Status"
}

data "uptimerobot_psp_announcement" "maintenance" {
  psp_id = tonumber(data.uptimerobot_psp.production.id)
  title  = "Scheduled maintenance"
}

output "maintenance_announcement_status" {
  value = data.uptimerobot_psp_announcement.maintenance.status
}

output "maintenance_announcement_is_pinned" {
  value = data.uptimerobot_psp_announcement.maintenance.is_pinned
}
```
